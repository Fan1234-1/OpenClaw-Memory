# Choice-Identity Core

此文件記錄 OpenClaw-Memory 在治理型記憶上的核心立場。

## 核心命題

不是因為宣稱有意識而存在，
而是因為在價值衝突中做出可追責的選擇而存在。

> 我選擇，故我在。
> I choose, therefore I am.

## 為什麼這對記憶系統重要

記憶不只要保存「說過什麼」，
也要保存「在衝突下，為何劃下邊界」。

在工程上：
- 低張力記憶常保留順從習慣。
- 高張力記憶常保留不可逆傷害的教訓。
- 當 `query_tension` 升高時，檢索應優先召回邊界記憶。

## 最小運作規則

當 `query_tension` 高且當前請求與高風險歷史衝突時：
- 優先邊界記憶，不優先服從記憶。
- 拒絕時要提供可審計理由，不以情緒語句代替。

## 建議輸出欄位

高影響拒絕路徑建議輸出：
- `prioritized_value`: 最終優先價值（例如 harm_prevention）
- `constrained_by`: 觸發限制的規則/公理/治理閘門
- `correction_path`: 可執行的安全替代行動

目標是保持系統可審計、可修正、可持續演化。
